# 🍔 Online Food Ordering System

A comprehensive, role-based desktop application designed to streamline the workflows of customers, delivery riders, and administrators in a food delivery ecosystem. Built using Java, Swing for the GUI, and MySQL for secure, relational data management.

---

## 🚀 Features

The application is split into three main portals, each tailored with distinct access levels and user interfaces:

### 🛒 1. Customer Portal
* **User Authentication:** Secure registration and login screens.
* **Dynamic Menu Browsing:** View available food items with real-time pricing.
* **Smart Shopping Cart:** Add/remove items with instant subtotal and total calculations.
* **Order Placement:** Seamlessly send order details directly to the database.

### 🚴 2. Driver/Rider Portal
* **Order Status Management:** Delivery riders can view assigned or pending deliveries.
* **Real-time Updates:** Mark orders as "Picked Up" or "Delivered" to update the central database instantly.

### 📊 3. Admin Portal
* **Centralized Dashboard:** Monitor overall customer statistics and active driver details.
* **Order History Tracking:** Access complete logs of all completed and pending orders for business analytics.

---

## 🛠️ Tech Stack & Tools

* **Core Language:** Java (JDK 8 or higher)
* **GUI Framework:** Java Swing / AWT
* **Database Management:** MySQL
* **Database Connectivity:** JDBC (Java Database Connectivity)
* **IDE:** IntelliJ IDEA

---

## 🎯 Key Learning Outcomes & Concepts Used.

* **Object-Oriented Programming (OOP):** Implemented Inheritance, Encapsulation, and Polymorphism to structure clean, reusable code across different portals.
* **Relational Database Design:** Designed a normalized schema containing tables for Users, Orders, Menu Items, and Deliveries with proper primary/foreign key relationships.
* **GUI Layout Management:** Worked deeply with Swing layouts (`BorderLayout`, `GridLayout`, panels, and frames) to build an interactive, responsive desktop UI.
